2 Tactics

2.1 eapply

Existential Apply: Allows you to leave a certain variable in a applied theorem undetermined until later.

2.2 omega

2.3 inversion

Break up an inductive type into it's constituent parts that were used to build it originally, bringing those hypotheses into the context.

2.4 auto

Solves goals that are solvable by any combination of intros, apply, reflexivity.

eauto: Same as auto but uses eapply instead of apply.

2.4.1 The Hint Database

auto uses local hypotheses as well as the global hint database for solving proofs. You can alter the hint database with the following commands:

  • Hint Resolve T. Add theorem or constructor T to the DB.
  • Hint Constructors C. Add all constructors of c to DB.
  • Hint Unfold d. Automatically expand defined symbol d during auto.
